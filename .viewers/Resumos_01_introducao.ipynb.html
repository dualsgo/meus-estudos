# Vamos explorar na prática os erros de sintaxe em Python.
# Erros de sintaxe são erros que ocorrem quando o código NÃO ESTÁ de acordo com as regras da linguagem de programação. 

# Imagine que deseja digitar o número 1000000 (um milhão), mas para melhorar a legibilidade do código, você deseja separar os milhares.

# Observação: O comando print() está comentado para evitar a interrupção da execução do código. Para descomentar o comando, basta remover o caractere #.

# print(1.000.000)
# SyntaxError: invalid syntax. Perhaps you forgot a comma?

# Tradução:
# Erros de Sintaxe: sintaxe inválida. Talvez você tenha esquecido de uma vírgula?

# O Python se esforça ao máximo para fornecer mensagens de erro claras e úteis. Nesse caso, a mensagem de erro indica que há um problema de sintaxe no código, sugerindo que talvez uma vírgula tenha sido esquecida. Nem sempre a sugestão estará correta, mas é uma tentativa do Python de ajudar o programador a identificar e corrigir o erro.

# Usamos pontos como separador, da forma que um humano faria e o Python gerou um erro de sintaxe, pois dentro das regras de sintaxe da linguagem o ponto é utilizado para representar números decimais. Portanto, o Python não consegue interpretar o número 1.000.000, pois não é um número decimal válido. Isso gerou uma mensagem de erro indicando que a sintaxe do código está incorreta e uma sugestão para corrigir o erro.

# Observação: Nesse exemplo, a mensagem de erro foi gerada, pois um número decimal válido possui apenas um ponto e utilizamos dois pontos. Sendo assim, é de se esperar que se removermos um dos pontos, o código será executado sem apresentar a mensagem de erro, mas o resultado não será o esperado.

# Removendo o primeiro ponto da direita para esquerda:
print(1.000000) # O resultado será 1.0

# Removendo o segundo ponto da direita para esquerda:
print(1000.000) # O resultado será 1000.0

# Vamos explorar na prática os erros semânticos em Python.

# Como vimos antes, imagine que deseja digitar o número 1000000 (um milhão), mas para melhorar a legibilidade do código, você deseja separar os milhares.

# Você pode digitar o número da seguinte forma:
# Obs.: Antes utilizamos pontos para separar os milhares, mas aqui usamos vírgulas para explorar o erro semântico.
print(1,000,000)

# Ao executar o código, o resultado será 1 0 0 que absolutamente não é o esperado.

# O resultado NÃO é 1,000,000, pois o Python interpreta o número 1,000,000 como três números separados por vírgulas, pois a vírgula é utilizada para separar argumentos em funções. Neste caso, a função print() recebe três argumentos: 1, 000 e 000. O Python exibe os argumentos separados por espaços, pois o separador de argumentos da função print() é um espaço em branco por padrão.

# Note que apesar do erro semântico, o Python não exibe uma mensagem de erro, pois o código é sintaticamente correto. O erro é apenas de lógica e o Python não pode adivinhar a intenção do programador. Portanto, é importante prestar atenção aos detalhes e testar o código para garantir que ele está funcionando corretamente.

# Erros semânticos são erros que ocorrem quando o código ESTÁ de acordo com as regras da linguagem de programação, mas o resultado não é o esperado.

# Este tipo de erro é chamado de erro semântico, pois o código é sintaticamente correto, mas o resultado não é o esperado. Python permite que você utilize a vírgula, mas não a interpreta como um separador de milhares. Para isso, você deve utilizar o caractere underscore (_) para separar os milhares se desejar melhorar a legibilidade do código.

# Veja o exemplo correto:
print(1_000_000) # O resultado será 1000000
1000000
